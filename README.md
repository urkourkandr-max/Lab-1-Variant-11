У цій лабораторній роботі реалізовано REST API веб-застосунок для керування користувачами та чергуваннями у лабораторії. Користувач може створювати нові записи, переглядати список усіх записів, отримувати окремий запис за його ID, редагувати існуючі записи, частково оновлювати поля та видаляти записи. Реалізовано серверну валідацію даних з виведенням повідомлень про помилки. Дані зберігаються у масивах JavaScript в оперативній пам’яті сервера, тому після його перезапуску очищуються. Реалізовано централізовану обробку помилок та логування всіх HTTP-запитів. Відповіді повертаються у форматі JSON без перезавантаження сервера

1 Запуск проеєкту

Відкрити папку backend. Відкрити термінал у цій папці. Виконати команду npm run dev. Після запуску у консолі повинно з’явитися повідомлення Server running on http://localhost:3000. Проєкт буде доступний за адресою http://localhost:3000

2 Перевірка валідації

Надіслати POST-запит з порожнім body ---- Повинно з’явитися повідомлення про помилку з вимогою заповнення обов’язкових полів

Передати поле неправильного типу (наприклад число замість рядка) ---- Повинна з’явитися помилка про неправильний тип даних

Передати email у неправильному форматі ---- Повинна з’явитися помилка про некоректний формат email

Ввести ім’я з одним символом ---- Повинна з’явитися помилка про мінімальну довжину поля

Ввести ім’я більше допустимої кількості символів ---- Повинна з’явитися помилка про перевищення максимальної довжини

Передати неіснуючий ID ---- Повинна з’явитися помилка 404 Not Found

3 Відображення даних

Після успішного створення запису сервер повинен повернути статус 201 Created та JSON створеного об’єкта з автоматично згенерованим ID

GET-запит до списку ---- Відображає всі створені записи у полі items

GET-запит за конкретним ID ---- Повертає лише один відповідний запис

PUT-запит ---- Повністю оновлює дані запису

PATCH-запит ---- Оновлює лише передані поля запису

DELETE-запит ---- Видаляє запис та повертає статус 204 No Content

4 Додаткового функціоналу

Логування: Після кожного запиту у консолі повинні з’являтися записи виду GET /api/users 200 - 3ms

Централізована обробка помилок: Усі помилки повертаються в однаковому JSON-форматі через error handler

Збереження даних: Дані зберігаються в оперативній пам’яті сервера та очищуються після його перезапуску.
